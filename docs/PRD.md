## 1. 项目背景

### 1.1 问题陈述

Windows 用户桌面在长期使用后不可避免地会积累大量快捷方式图标（`.lnk` 和 `.url` 文件），包括应用程序、文件、文件夹和网页链接。这些图标缺乏有效的分类机制，导致桌面杂乱无章，用户难以快速定位所需应用。

现有的桌面整理工具存在以下不足：
- **侵入性强**：部分工具替换 Windows 原生桌面行为，导致兼容性问题
- **功能臃肿**：集成大量非核心功能，占用系统资源
- **设计落后**：界面风格与现代操作系统不匹配
- **数据不安全**：配置存储在系统目录，卸载后残留数据

### 1.2 产品定位

桌面图标收纳盒是一款 **轻量级、非侵入性** 的 Windows 桌面快捷方式管理工具。它不对 Windows 桌面进行任何修改，而是提供一个独立的管理界面，让用户通过"收纳盒"（分类箱）的概念对快捷方式进行分组整理，并可将常用分类以透明浮窗形式固定在桌面上。

### 1.3 产品目标

| 目标 | 描述 |
|------|------|
| **整洁桌面** | 帮助用户将杂乱的桌面快捷方式分类整理到自定义收纳盒中 |
| **快速访问** | 通过桌面浮窗提供常用应用的即时访问能力 |
| **零侵入** | 不修改 Windows 桌面行为，所有数据可便携存储 |
| **美观体验** | 提供现代感的玻璃拟态（Glass Morphism）暗色主题界面 |

## 2. 目标用户

### 2.1 用户画像

| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| **效率工作者** | 桌面快捷方式超过 20 个，频繁使用开发工具和办公软件 | 快速分类、一键启动、减少寻找时间 |
| **开发者** | 安装大量开发工具、编辑器、终端，桌面图标密集 | 按项目/工具类型分组，浮窗快速切换 |
| **普通 Windows 用户** | 桌面长期未整理，图标混杂难以管理 | 简单易用的分类功能，可视化管理 |
| **追求美观的用户** | 不满 Windows 默认桌面样式，追求个性化桌面 | 透明浮窗、玻璃拟态主题、自定义图标颜色 |

### 2.2 使用场景

- **场景 1**：开发者安装了 VS Code、IntelliJ IDEA、Terminal 等工具，希望按"开发工具"分类管理
- **场景 2**：用户同时使用多个浏览器（Chrome、Firefox、Edge），希望将它们归入"浏览器"盒子
- **场景 3**：用户经常使用微信、钉钉、飞书等通讯工具，希望在桌面固定一个"通讯"浮窗
- **场景 4**：用户希望一键整理所有未分类的快捷方式到对应的盒子中

### 2.3 环境要求

- **操作系统**：Windows 10 / Windows 11
- **运行时依赖**：PowerShell（系统内置）
- **硬件要求**：无特殊要求，普通 PC 即可运行

## 3. 产品概述

### 3.1 核心概念

| 概念 | 定义 |
|------|------|
| **收纳盒（Box）** | 一个命名的分类容器，用于存放同一类别的快捷方式 |
| **未分类区（Unassigned）** | 存放尚未归入任何盒子的桌面快捷方式 |
| **浮窗（Desktop Widget）** | 固定在桌面上的透明小窗口，实时显示某个盒子的内容 |
| **快速整理（Quick Organize）** | 基于关键词匹配的自动分类功能 |

## 5. 详细功能说明

### 5.1 快捷方式扫描（F-01 ~ F-03）

**功能描述**：自动扫描 Windows 桌面上的所有 `.lnk` 和 `.url` 快捷方式文件。

**扫描范围**：
- 用户桌面目录：`%USERPROFILE%\Desktop`
- 公共桌面目录：`C:\Users\Public\Desktop`

**处理流程**：
1. 遍历两个桌面目录，收集所有 `.lnk` 和 `.url` 文件
2. 对于 `.lnk` 文件：通过 PowerShell 脚本（`parse-lnk.ps1`）使用 `WScript.Shell` COM 对象解析，提取目标路径（TargetPath）、图标路径（IconLocation）、工作目录（WorkingDirectory）、启动参数（Arguments）
3. 对于 `.url` 文件：直接解析 INI 格式内容，提取 `URL=` 和 `IconFile=` 字段
4. 返回完整的快捷方式列表，包含名称、路径、类型、图标路径等信息

**数据输出格式**：
```json
{
  "name": "VS Code",
  "path": "C:\\Users\\xxx\\Desktop\\Visual Studio Code.lnk",
  "type": "lnk",
  "targetPath": "C:\\Program Files\\Microsoft VS Code\\Code.exe",
  "iconPath": "C:\\...\\Code.exe",
  "workingDir": "C:\\Program Files\\Microsoft VS Code",
  "arguments": ""
}
```

### 5.2 分类管理（F-04 ~ F-08）

**创建收纳盒**：
- 用户输入盒子名称（必填）
- 选择图标：提供 8 个预设表情符号（🎮 📱 💻 📁 🔧 🎬 🌐 📝）
- 选择颜色：提供 6 种色调（蓝、绿、紫、橙、红、粉）
- 选择显示模式：面板模式（仅管理窗口内显示）或 桌面模式（同时创建浮窗）

**拖拽移动**：
- 支持从未分类区拖拽到任意盒子
- 支持从盒子拖拽回未分类区
- 支持盒子之间互相拖拽
- 支持从主窗口拖拽到桌面浮窗
- 支持从桌面浮窗拖拽到其他浮窗

**右键菜单**：
- 打开快捷方式
- 打开文件所在位置
- 移动到...（子菜单列出所有盒子）
- 从盒子中移除 → **操作结果**：将快捷方式从当前盒子中移除，回退到未分类区（不是从应用中彻底删除）

**删除收纳盒**：
- 点击删除时弹出二次确认对话框，提示："确定要删除盒子「{盒子名}」吗？盒子内的 {N} 个快捷方式将回到未分类区。"
- 确认后：盒子内所有快捷方式回到未分类区，盒子对应的浮窗自动关闭，`hiddenItems` 中与该盒子关联的隐藏记录保留（不影响已隐藏文件的状态）
- 取消则不执行任何操作

**显示模式切换**：
- 创建后可通过右键盒子标题栏 → "切换显示模式"在面板模式和桌面模式之间切换
- `displayMode` 与 `onDesktop` 的关系：`onDesktop` 是运行时状态（浮窗是否实际打开），`displayMode` 是用户的偏好设置
  - 设置 `displayMode="desktop"` 时：自动设置 `onDesktop=true` 并创建浮窗
  - 关闭浮窗（`onDesktop=false`）：`displayMode` 保持不变，下次打开主窗口时可重新开启
  - 设置 `displayMode="panel"` 时：自动关闭浮窗并设置 `onDesktop=false`

### 5.3 图标提取（F-09 ~ F-12）

**提取策略**（按优先级）：
1. **磁盘缓存命中**：检查 `icons/` 目录中是否存在 MD5 命名的 PNG 文件，且源文件修改时间未变化 → 直接返回
2. **内存缓存命中**：检查内存中是否存在有效缓存 → 直接返回
3. **PowerShell 提取**：通过 `extract-icon.ps1` 脚本提取：
   - `.ico` 文件：直接通过 `System.Drawing.Icon` 读取
   - `.exe`/`.dll` 文件：通过 `Icon.ExtractAssociatedIcon()` 提取
   - 其他文件（`.lnk`/`.cmd`/`.bat`）：先尝试 `ExtractAssociatedIcon`，失败后回退到 Win32 `SHGetFileInfo` Shell API
4. **提取失败**：显示回退表情（🌐 或 📁）

**并发提取**：
- 默认 4 个并发工作线程
- 每完成一个图标提取，立即通过 IPC 推送 `icon-updated` 事件到渲染进程
- 渲染进程增量更新 DOM，无需全量重渲染

### 5.4 桌面浮窗（F-13 ~ F-16）

**窗口特性**：
- 无边框透明窗口（`frame: false, transparent: true`）
- 始终置顶（`alwaysOnTop: true`）
- 可拖拽定位（标题栏拖拽）
- 可调整大小
- 可折叠/展开内容区
- 位置和大小自动保存到配置文件

**多显示器与位置修正**：
- 创建浮窗时，位置默认为当前显示器的中心区域
- 每次显示浮窗前，检查 `desktopPos` 对应的坐标是否在当前任一显示器的可见区域内
- 若坐标超出所有显示器范围（如用户断开外接显示器后），自动将浮窗重置到主显示器的中心位置
- 用户可通过右键标题栏 →"重置位置"手动触发重置

**多浮窗布局**：
- 新创建的浮窗自动偏移排列，避免与已有浮窗重叠
- 用户拖拽浮窗时，自动检测与其他浮窗的重叠区域并进行微调避让

**交互能力**：
- 从主窗口拖拽快捷方式到浮窗
- 从其他浮窗拖拽快捷方式到本浮窗
- 从 Windows 资源管理器直接拖拽 `.lnk`/`.url` 文件到浮窗
- 右键菜单（打开、打开文件位置、从盒子移除）
- 关闭浮窗不删除盒子数据，可从主窗口重新打开

### 5.5 快速整理（F-17）

**算法逻辑**：
1. 遍历所有未分类的快捷方式
2. 对每个快捷方式，计算其与所有盒子的**匹配分数**：
   - **精确匹配**（快捷方式名称完全等于盒子名称，不区分大小写）：得分 3
   - **盒子名包含快捷方式名**（如盒子"浏览器"匹配快捷方式"Chrome"→ 否；盒子"浏览器"匹配快捷方式"浏览器收藏"→ 是）：得分 2
   - **快捷方式名包含盒子名**（如快捷方式"开发工具箱"匹配盒子"工具"→ 是）：得分 1
   - **无匹配**：得分 0
3. **多匹配冲突处理**：
   - 若唯一最高分 ≥ 3：自动归入最高分盒子（仅精确匹配自动归入）
   - 若最高分为 2 且仅一个盒子匹配：**标记为候选建议**，不自动归入（子串匹配存在歧义风险，需用户确认）
   - 若最高分为 1 且仅一个盒子匹配：**标记为候选建议**
   - 若最高分 ≥ 2 且多个盒子匹配：跳过（模糊匹配多义冲突）
   - 若所有盒子得分均为 0：留在未分类区
4. **候选建议处理**：候选项在 UI 中以虚线边框标记，用户可一键确认归入建议的盒子，或手动拖拽到其他位置
5. 记录整理时间到 `config.lastOrganizeTime`
6. 整理完成后弹出 Toast 通知，显示汇总结果，例如："快速整理完成：已归类 12 个项目到 4 个盒子，跳过 3 个模糊匹配项"

**设计原则**：宁可漏分类，不可误分类。对于模糊匹配（子串包含但非精确匹配）的场景，优先保守处理。

**触发方式**：
- 主窗口工具栏的快速整理按钮（⚡ 图标）
- 键盘快捷键 `Ctrl+Shift+O`（仅主窗口焦点时生效，非全局）
- 系统托盘菜单中的"快速整理"选项

### 5.6 桌面图标隐藏（F-19 ~ F-20）

**隐藏机制**：
1. 将已收纳到盒子中的 `.lnk`/`.url` 文件从桌面移动到备份目录 `icons/shortcuts/`
2. 在 `config.hiddenItems[]` 中记录原始路径和备份路径的映射关系
3. 启动时如果设置为隐藏状态，自动执行隐藏操作

**权限处理**：
- 用户桌面目录（`%USERPROFILE%\Desktop`）：普通权限即可读写，无需权限提升
- 公共桌面目录（`C:\Users\Public\Desktop`）：需要管理员权限。处理策略：
  1. 检测目标文件是否位于公共桌面
  2. 若是，尝试直接移动；若因权限不足失败，弹出提示："公共桌面快捷方式需要管理员权限才能隐藏，请右键以管理员身份运行"
  3. 提供"以管理员身份重启"按钮，通过 `ShellExecute` 的 `runas` 动词重新启动应用
  4. 若用户拒绝提权，跳过该文件，仅隐藏用户桌面的快捷方式

**隐藏状态下的扫描行为**：
- 每次启动时，仅扫描原始桌面目录（用户桌面 + 公共桌面），**不扫描备份目录**
- 同步逻辑（`syncUnassigned`）会检测：如果某个快捷方式的原始路径存在于 `hiddenItems` 中，则将其从扫描结果中标记为"已隐藏"，不会重复添加到 unassigned 区
- 新安装的程序在桌面创建的快捷方式会被正常发现并添加到 unassigned 区

**恢复机制**：
1. 从备份目录将文件移回原始桌面位置
2. 清除 `config.hiddenItems[]` 中的对应记录

**隐藏状态下从盒子移出的文件处理**：

场景：快捷方式已被隐藏（文件在 `icons/shortcuts/` 备份目录中），用户将其从盒子拖回未分类区。

| 步骤 | 行为 |
|------|------|
| 1 | 检测该快捷方式是否在 `hiddenItems[]` 中 |
| 2 | 如果在 → 判断原始路径是否位于公共桌面 |
| 2a | 用户桌面 → 直接将文件从备份目录移回原始位置 |
| 2b | 公共桌面 → 尝试移动；若权限不足，弹出提示并提供"以管理员身份重启"选项（与隐藏时的权限策略对称） |
| 3 | 清除 `hiddenItems[]` 中的对应记录 |
| 4 | 将快捷方式添加到 `config.unassigned[]` |
| 5 | UI 显示该快捷方式出现在未分类区（桌面图标也同步恢复可见） |

**设计意图**：从盒子移出 = 用户不再需要该快捷方式被收纳管理 = 应该让它回到桌面原位。隐藏和移除是两个独立操作，但隐藏状态下的移除应触发自动恢复。

**注意**：此功能仅移动已收纳到盒子中的快捷方式，未分类的快捷方式不受影响。

### 5.7 活动日志（F-21）

**记录的事件类型**：
| 事件类型 | 说明 |
|----------|------|
| move | 快捷方式在盒子之间或与未分类区之间移动 |
| create | 创建新的收纳盒 |
| delete | 删除收纳盒 |
| rename | 重命名收纳盒 |
| quick-organize | 执行快速整理 |
| hide-icons | 隐藏桌面图标 |
| show-icons | 显示桌面图标 |

**存储限制**：默认最多保留 200 条日志记录，超出后自动清除最早的记录。用户可在设置面板中调整上限（100 / 200 / 500 / 1000），修改后立即生效（裁剪多余记录）。

**清除功能**：
- UI 入口：活动日志面板底部提供"清除日志"按钮
- 清除粒度：一键清空全部日志记录
- 执行前弹出二次确认："确定要清除所有活动日志吗？此操作不可撤销。"

**导出功能**：
- UI 入口：活动日志面板底部提供"导出日志"按钮
- 导出格式：CSV 文件（含时间戳、事件类型、描述、详情），可通过 Excel 打开
- 导出范围：全部日志记录
- 文件命名：`desktop-organizer-log-{YYYYMMDD-HHmmss}.csv`

**时间格式**：
- 状态栏和托盘菜单中显示相对时间（如"刚刚"、"5分钟前"、"2小时前"、"3天前"）
- 活动日志面板中显示绝对时间（如"2026-06-06 14:30:25"）
- `lastOrganizeTime` 仅在快速整理时更新，手动拖拽分类不更新此字段

### 5.8 系统托盘（F-22）

**托盘菜单内容**：
- 显示主窗口
- 快速整理（显示未分类数量，如"快速整理 (5)"）
- 退出

**行为**：
- 关闭主窗口时隐藏到托盘而非退出
- 托盘菜单每 30 秒自动刷新
- 双击托盘图标显示主窗口

### 5.9 搜索（F-18）

**搜索范围**：全局搜索，覆盖所有盒子中的快捷方式以及未分类区的快捷方式。

**搜索维度**：
- 快捷方式名称（主要维度）
- 快捷方式目标路径（辅助维度，如搜索 "Code" 可匹配目标路径含 "Code.exe" 的项目）

**交互行为**：
- 输入即搜索（debounce 200ms），无需按回车
- 搜索结果以高亮方式显示在原有位置（不改变布局），未匹配的项目降低透明度
- 按 `Escape` 或清空搜索框恢复全部显示
- 搜索框位于工具栏，通过 `Ctrl+F` 聚焦

### 5.10 键盘快捷键作用域（F-24）

所有快捷键**仅在主管理窗口获得焦点时生效**，不注册为全局快捷键，避免干扰其他应用：

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+F` | 聚焦搜索框 | 与大多数应用行为一致 |
| `Ctrl+N` | 创建新盒子 | 打开创建对话框 |
| `Ctrl+R` | 刷新桌面快捷方式 | 重新扫描桌面并更新列表 |
| `Ctrl+Shift+O` | 快速整理 | 执行自动分类 |
| `Escape` | 关闭弹窗/搜索 | 关闭当前打开的对话框或清除搜索 |

### 5.11 撤销操作（F-06a）

**支持的撤销场景**：
- 快捷方式移动（盒子间拖拽、从未分类区拖入、从盒子拖出）
- 盒子删除（连带恢复盒子内所有项目到未分类区）

**交互入口**：
- **快捷键**：`Ctrl+Z`（仅主窗口焦点时生效）
- **Toast 操作按钮**：每次移动/删除操作后，Toast 通知右侧附带"撤销"按钮

**时效限制**：撤销操作的有效窗口为 **30 秒**。超过 30 秒后撤销按钮消失，`Ctrl+Z` 不再响应上一次操作。

**快速整理的撤销**：
- 快速整理一次性移动多个快捷方式时，撤销作为**整体回退**——将所有被自动归类的项目一次性移回未分类区
- 不支持分步撤销（如只撤销其中某一个项目的归类）

**删除盒子的撤销**：
- 删除盒子后 30 秒内可通过 `Ctrl+Z` 或 Toast 按钮撤销
- 撤销后恢复盒子及其所有项目，浮窗状态（`onDesktop`/`desktopPos`）同步恢复

**实现机制**：主进程维护一个最近操作栈（`undoStack`），仅保留最近 1 条操作的完整快照（操作前的 `boxes` + `unassigned` + `hiddenItems` 快照），新操作入栈时清空旧快照。

### 5.12 配置导入/导出（F-29）

**导出**：
- UI 入口：设置面板 → "导出配置"按钮
- 导出内容：`config.json` 完整内容（含 `iconData`）
- 文件格式：`.json` 文件，文件名格式 `desktop-organizer-config-{YYYYMMDD}.json`
- 文件大小提示：导出前显示预估文件大小（基于 `iconData` 总量），若超过 10MB 弹窗确认

**导入**：
- UI 入口：设置面板 → "导入配置"按钮
- 导入流程：
  1. 用户选择 `.json` 配置文件
  2. 系统解析并验证文件格式（JSON 语法 + 必要字段检查）
  3. **路径差异处理**：不自动替换路径。导入后如果快捷方式的原始路径在当前系统不存在（如不同用户名），在 UI 中标记为"路径失效"，用户可手动修正或删除
  4. **同名盒子冲突**：如果本地已有同名盒子，弹窗提供三个选项：
     - **合并**：将导入的项目追加到现有盒子中
     - **覆盖**：用导入的盒子完全替换现有盒子（项目和设置）
     - **跳过**：保留现有盒子，忽略导入的同名盒子
  5. 确认后覆盖本地 `config.json`，重启应用生效

### 5.13 排序功能（F-30）

**排序入口**：
- 盒子列表排序：工具栏"排序"下拉菜单（或右键盒子区域空白处）
- 盒子内快捷方式排序：右键盒子标题栏 → "排序方式"

**排序模式**：
| 模式 | 说明 |
|------|------|
| 手动排序 | 用户通过拖拽调整顺序，手动排序后自动切换为"自定义"模式 |
| 字母序 | 按名称 A-Z 排序（中文按拼音首字母） |
| 创建时间 | 按快捷方式添加到盒子的时间排序（最新在前） |
| 最近使用 | 按最近一次启动的时间排序（最常用在前） |

**作用范围**：排序按盒子独立设置，每个盒子可有不同的排序偏好。

**持久化**：排序偏好保存在每个 Box 对象的 `sortMode` 字段中，手动排序时同时保存 `sortOrder` 数组（记录快捷方式路径顺序）。

### 5.14 无效快捷方式检测（F-34）

**检测时机**：
- **首次扫描**：应用启动时，快捷方式扫描完成后，异步执行目标路径存在性检测
- **延迟执行**：不阻塞启动流程，先显示所有快捷方式（图标和名称），检测结果异步更新
- **性能保护**：最多同时检测 10 个项目（并发控制），避免大量文件系统 IO 导致卡顿

**UI 标识**：
- 无效快捷方式在卡片上显示灰色蒙层 + ⚠️ 图标
- 鼠标悬停提示"目标程序不存在，可能已被卸载"
- 盒子标题栏显示无效项计数（如"开发工具 (2⚠️)"）

**批量清理**：
- 设置面板中提供"清理无效快捷方式"按钮
- 点击后列出所有无效项，用户可逐个确认或一键全部移除
- 移除操作可撤销（30 秒内）

## 6. 用户流程

### 6.1 首次启动流程

```
启动应用
  │
  ├── 单实例检查 ──→ 已有实例运行？──→ 聚焦已有窗口 ──→ 结束
  │
  ├── 扫描桌面快捷方式
  │     ├── 用户桌面
  │     └── 公共桌面
  │
  ├── 解析所有 .lnk/.url 文件
  │
  ├── 并发提取图标（4线程）
  │     └── 每完成一个 → 推送到渲染进程增量更新
  │
  ├── 加载配置文件
  │     └── 文件不存在 → 创建默认配置
  │
  ├── 同步未分类区
  │     └── 将新发现的快捷方式添加到 unassigned
  │
  ├── 隐藏已收纳的桌面图标（如果设置启用）
  │
  ├── 创建系统托盘
  │
  └── 显示主窗口（960×700，暗色背景）
```

### 6.2 创建收纳盒流程

```
用户点击"创建"按钮
  │
  ├── 弹出创建对话框
  │     ├── 输入盒子名称
  │     ├── 选择图标（8个预设表情）
  │     ├── 选择颜色（6种色调）
  │     └── 选择显示模式
  │           ├── 面板模式 ──→ 仅在管理窗口显示
  │           └── 桌面模式 ──→ 同时创建桌面浮窗
  │
  ├── 保存到 config.boxes[]
  │
  ├── 记录活动日志
  │
  ├── 通知所有桌面浮窗更新
  │
  └── 重新渲染界面
```

### 6.3 拖拽分类流程

```
用户拖拽快捷方式卡片
  │
  ├── 拖拽开始
  │     └── 设置 dragData（来源 + 路径）
  │
  ├── 拖拽到目标区域
  │     ├── 目标是盒子 → handleDropOnBox()
  │     │     ├── 从 unassigned 移除
  │     │     ├── 添加到目标盒子 items[]
  │     │     └── 保存配置
  │     │
  │     ├── 目标是未分类区 → handleDropOnUnassigned()
  │     │     ├── 从源盒子移除
  │     │     └── 添加回 unassigned
  │     │
  │     └── 目标是桌面浮窗
  │           ├── Electron 跨窗口拖拽通过 IPC 中转实现：
  │           │     ├── 源窗口（主窗口或浮窗）设置 dragData 并触发拖拽
  │           │     ├── 用户将拖拽元素释放到目标浮窗
  │           │     ├── 目标浮窗的 drop 事件读取数据
  │           │     └── 通过 ipcRenderer.invoke('desktop-box:add-item', ...) 
  │           │         将数据发送到主进程处理
  │           └── 主进程更新配置后通知所有相关窗口刷新
  │
  ├── 记录活动日志
  │
  └── 重新渲染界面
```

### 6.4 快速整理流程

```
用户触发快速整理
  │
  ├── 读取未分类列表
  │
  ├── 对每个未分类快捷方式：
  │     ├── 遍历所有盒子，计算匹配分数：
  │     │     ├── 精确匹配（名称完全相等） → 得分 3
  │     │     ├── 盒子名包含快捷方式名 → 得分 2
  │     │     ├── 快捷方式名包含盒子名 → 得分 1
  │     │     └── 无匹配 → 得分 0
  │     │
  │     ├── 分数决策：
  │     │     ├── 唯一最高分 ≥ 3 → 自动归入该盒子
  │     │     ├── 最高分 = 2 且仅一个盒子 → 自动归入该盒子
  │     │     ├── 最高分 = 2 且多个盒子 → 跳过（模糊匹配）
  │     │     ├── 最高分 = 1 → 跳过（仅子串包含，不自动归类）
  │     │     └── 全部为 0 → 留在未分类区
  │     │
  │     └── 自动归入 → 从未分类区移除，添加到目标盒子
  │
  ├── 更新 lastOrganizeTime
  │
  ├── 记录活动日志
  │
  ├── 生成整理结果：
  │     ├── 计数：已归类 N 个项目到 M 个盒子
  │     ├── 记录：跳过项列表（名称 + 原因）
  │     └── 写入活动日志的 quick-organize 事件 detail 字段
  │
  └── 显示 Toast 通知：
        ├── "已归类 12 个项目到 4 个盒子"
        ├── 若有跳过项：追加"，跳过 3 个模糊匹配项"
        └── Toast 内含可点击的"查看详情"链接 → 打开活动日志面板
```

### 6.5 启动/打开快捷方式流程

```
用户双击快捷方式卡片
  │
  ├── 尝试启动（按优先级）：
  │     ├── 1. 检查原始路径是否存在
  │     │     ├── 存在 → 直接使用原始路径启动
  │     │     └── 不存在 → 检查是否在 hiddenItems 中
  │     │           ├── 在 → 使用备份路径（backupPath）直接启动

  │     └── 3. 目标程序被卸载的处理：
  │           ├── 检测到目标路径不存在

  └── 启动失败 → 显示错误提示（包含具体失败原因）
```

## 7. 非功能需求

### 7.1 性能需求

| 指标 | 要求 |
|------|------|
| 启动时间 | 冷启动到界面可交互 < 3 秒 |
| 图标提取 | 首批图标（前 10 个）应在 2 秒内显示 |
| 拖拽响应 | 拖拽操作响应延迟 < 100ms |
| 内存占用 | 空闲状态内存占用 < 150MB |
| 配置保存 | 使用防抖（300ms）避免频繁磁盘写入 |
| 图标缓存 | 二次启动时图标应从缓存直接加载，无需重新提取 |

### 7.2 可靠性需求

| 指标 | 要求 |
|------|------|
| 数据持久化 | 配置使用原子写入（先写 `.tmp` 再重命名），防止写入中断导致数据损坏 |
| 错误恢复 | 图标提取失败时显示回退表情，不影响其他功能 |
| 单实例 | 同一时间只能运行一个实例，第二次启动聚焦已有窗口 |
| 向后兼容 | 配置文件格式升级时自动修复旧版本格式 |
| 进程隔离 | 渲染进程崩溃不影响主进程和其他浮窗 |

### 7.3 安全需求

| 指标 | 要求 |
|------|------|
| 上下文隔离 | `contextIsolation: true`，渲染进程无法直接访问 Node.js API |
| 预加载脚本 | 通过 `contextBridge.exposeInMainWorld` 暴露最小化的 API |
| 脚本执行 | PowerShell 脚本执行设置 10 秒超时，防止挂起 |
| XSS 防护 | 所有用户输入通过 `escapeHtml()` 和 `escapeAttr()` 转义 |

### 7.4 可用性需求

| 指标 | 要求 |
|------|------|
| 界面语言 | 中文界面（简体） |
| 无障碍 | 支持键盘快捷键操作主要功能 |
| 错误提示 | 操作失败时显示 Toast 通知 |
| 状态反馈 | 状态栏实时显示系统信息和操作状态 |

### 7.5 可维护性需求

| 指标 | 要求 |
|------|------|

## 9. 数据模型

### 9.1 配置文件结构（config.json）

```json
{
  "version": 2,
  "boxes": [
    {
      "id": "box_1717500000000_abc123",
      "name": "开发工具",
      "items": [
        {
          "name": "Visual Studio Code",
          "path": "C:\\Users\\xxx\\Desktop\\Visual Studio Code.lnk",
          "type": "lnk",
          "targetPath": "C:\\Program Files\\Microsoft VS Code\\Code.exe",
          "iconPath": "C:\\Program Files\\Microsoft VS Code\\Code.exe",
          "iconData": "base64编码的PNG数据..."
        }
      ],
      "onDesktop": false,
      "collapsed": false,
      "desktopPos": { "x": 100, "y": 100 },
      "desktopSize": { "width": 260, "height": 320 },
      "icon": "💻",
      "color": "#4A9EFF",
      "displayMode": "panel",
      "sortMode": "manual",
      "sortOrder": [
        "C:\\Users\\xxx\\Desktop\\Visual Studio Code.lnk",
        "C:\\Users\\xxx\\Desktop\\GitHub Desktop.lnk"
      ]
    }
  ],
  "unassigned": [],
  "hideCollectedIcons": false,
  "hiddenItems": [],
  "lastOrganizeTime": 1717500000000
}
```

### 9.2 字段说明

**顶级字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `version` | Number | 配置格式版本号（当前值：`2`），用于升级迁移检测 |
| `boxes` | Array\<Box\> | 所有收纳盒的列表 |
| `unassigned` | Array\<Shortcut\> | 未分类的快捷方式列表 |
| `hideCollectedIcons` | Boolean | 是否隐藏已收纳的桌面图标 |
| `hiddenItems` | Array\<HiddenItem\> | 已隐藏的快捷方式备份记录 |
| `lastOrganizeTime` | Number | 上次快速整理的时间戳（毫秒） |

**图标数据存储关系说明**：

应用采用两级图标缓存架构，`iconData` 与 `icons/` 磁盘缓存的关系如下：
1. **磁盘缓存（`icons/` 目录）**：主缓存。以源文件路径的 MD5 哈希命名为 PNG 文件，基于源文件修改时间（mtime）验证有效性
2. **内存缓存**：热缓存。应用运行期间保留在内存中的图标数据，避免重复磁盘读取
3. **`iconData` 字段**：兜底缓存。存储在 `config.json` 中的 Base64 编码 PNG 数据，主要作用：
   - 当磁盘缓存目录被意外清除或损坏时，仍可显示图标
   - 便携化场景下，`config.json` 可随 U 盘携带，确保图标在其他设备上可用
4. **写入时机**：图标提取成功后同时写入磁盘缓存和 `iconData` 字段
5. **读取优先级**：内存缓存 → 磁盘缓存 → `iconData` → 重新提取

**注意**：`iconData` 会导致 `config.json` 文件体积增大。对于 100+ 个快捷方式的场景，文件可能达到数 MB。优化方向：未来版本可考虑将 `iconData` 迁移到独立的图标索引文件，或仅对无法从磁盘缓存恢复的图标保留 `iconData`。

**Box 对象**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 唯一标识符（格式：`box_{timestamp}_{random}`） |
| `name` | String | 盒子名称 |
| `items` | Array\<Shortcut\> | 盒子中的快捷方式列表 |
| `onDesktop` | Boolean | 是否在桌面上显示浮窗 |
| `collapsed` | Boolean | 浮窗是否处于折叠状态 |
| `desktopPos` | Object | 浮窗位置 `{x, y}` |
| `desktopSize` | Object | 浮窗大小 `{width, height}` |
| `icon` | String | 盒子图标（表情符号） |
| `color` | String | 盒子主题色（十六进制） |
| `displayMode` | String | 显示模式：`"panel"` 或 `"desktop"` |
| `sortMode` | String | 排序模式：`"manual"` / `"alpha"` / `"createTime"` / `"recentUse"`，默认 `"manual"`（新盒子默认保持用户拖拽顺序） |
| `sortOrder` | Array\<String\> | 手动排序时的快捷方式路径（`path`）顺序数组，仅 `sortMode="manual"` 时有效 |

**Shortcut 对象**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | String | 快捷方式名称（不含扩展名） |
| `path` | String | 快捷方式文件的完整路径 |
| `type` | String | 文件类型：`"lnk"` 或 `"url"` |
| `targetPath` | String | 目标程序路径（仅 .lnk） |
| `iconPath` | String | 图标文件路径 |
| `iconData` | String | Base64 编码的图标 PNG 数据（**仅作为离线兜底**，正常流程从 `icons/` 磁盘缓存读取） |
| `workingDir` | String | 工作目录（仅 .lnk） |
| `arguments` | String | 启动参数（仅 .lnk） |
| `url` | String | URL 地址（仅 .url） |

**HiddenItem 对象**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `originalPath` | String | 原始桌面路径 |
| `backupPath` | String | 备份目录中的路径 |

## 12. 边缘场景与兜底策略

### 12.1 多显示器适配

| 场景 | 处理策略 |
|------|----------|
| 浮窗创建 | 跟随鼠标光标所在的显示器，在该显示器中心区域偏移显示（避免与其他浮窗重叠） |
| 外接显示器断开 | 启动时检测浮窗坐标是否在可见区域，不在则重置到主显示器中心 |
| 多显示器间拖拽浮窗 | 支持跨显示器边界拖拽，位置坐标基于虚拟屏幕全局坐标系 |
| 主窗口打开位置 | 默认在屏幕中心，记忆上次关闭时的位置 |

### 12.2 配置文件损坏恢复

| 场景 | 处理策略 |
|------|----------|
| `config.json` 语法错误（JSON 解析失败） | 自动备份损坏文件为 `config.json.bak`，使用默认空配置启动，Toast 提示"配置文件已损坏，已重置为默认配置。点击此处从备份恢复"——Toast 中的"从备份恢复"为可点击链接，点击后将 `.bak` 文件重命名回 `config.json` 并重启应用 |
| `config.json` 被外部篡改（字段缺失） | `loadConfig()` 中的向后兼容逻辑自动补全缺失字段（如缺少 `hiddenItems` 自动初始化为空数组） |
| 原子写入中断（`.tmp` 文件残留） | 启动时检测 `.tmp` 残留文件，若 `config.json` 不存在则用 `.tmp` 重命名恢复 |
| 配置文件被锁定（如被其他进程占用） | 保存失败时 Toast 提示"配置保存失败，请检查是否有其他程序占用"，内存中保留最新状态，下次操作时重试 |

### 12.3 PowerShell 脚本错误处理

| 场景 | 处理策略 |
|------|----------|
| `parse-lnk.ps1` 执行超时（>10秒） | 终止该进程，将该快捷方式标记为"解析失败"，在 UI 中显示为灰色占位符，不影响其他快捷方式的解析 |
| `parse-lnk.ps1` 执行异常 | 捕获 stderr 输出，记录到 `app.log`，该快捷方式使用文件名作为名称，跳过目标路径/图标提取 |
| `extract-icon.ps1` 执行超时 | 终止该进程，使用回退表情（🌐），后续启动时自动重试 |
| `extract-icon.ps1` 依赖的 COM 对象不可用 | 降级到基础图标提取（仅对 .ico 文件有效），其他文件显示回退表情 |
| PowerShell 被组策略禁用 | 启动时检测 PowerShell 可用性，若不可用则弹窗提示"此应用需要 PowerShell 支持，请联系系统管理员"，功能降级为仅显示文件名（无图标） |

### 12.4 无效快捷方式处理

| 场景 | 处理策略 |
|------|----------|
| 目标程序被卸载 | 双击启动时检测到目标不存在，Toast 提示"目标程序不存在，可能已被卸载"，快捷方式在 UI 中标记为无效（灰色+⚠️） |
| 快捷方式文件被删除 | `syncUnassigned` 同步时从列表中移除，自动清理 |
| 快捷方式指向网络驱动器（不可用） | 尝试启动失败后标记为无效，不阻塞其他操作 |
| 批量清理无效项 | 在设置面板中提供"清理无效快捷方式"按钮，一键移除所有目标不存在的快捷方式 |

### 12.5 浮窗与主窗口数据同步

| 操作 | 同步方式 | 延迟上限 |
|------|----------|----------|
| 主窗口修改盒子内容 | IPC 推送 `desktop-box:data-updated` 事件 | < 200ms |
| 浮窗修改盒子内容 | 通过 `desktop-box:remove-item`/`add-item` → 主进程处理 → 推送更新 | < 300ms |
| 主窗口和浮窗同时操作同一快捷方式 | 主进程串行处理 IPC 请求，避免文件级并发冲突。但业务语义冲突（同一快捷方式被同时移动到不同盒子）的处理：**先到先得**——第一个操作成功执行，第二个操作在执行时检测到源项目已不存在，返回 `{success: false, reason: "item_not_found"}`，渲染进程显示 Toast 提示"该快捷方式已被移动到其他位置"并自动刷新列表 | 依赖队列深度 |
| 浮窗窗口关闭 | 仅关闭窗口，不修改盒子数据 | 即时 |
| 主窗口最小化到托盘 | 浮窗不受影响，继续正常工作 | — |

### 12.6 升级迁移策略

| 版本变化 | 迁移策略 |
|----------|----------|
| 小版本升级（如 v1.3.0 → v1.3.1） | `loadConfig()` 中的向后兼容逻辑自动处理，无需用户干预 |
| 大版本升级（如 v1.x → v2.0） | 在 `loadConfig()` 中增加版本号检测（读取 `config.version` 字段），执行对应的迁移脚本 |
| 迁移失败 | 自动备份原配置为 `config.json.pre-migration`，使用兼容模式启动 |
| 配置版本号缺失 | 视为最早版本，执行全量迁移路径 |

**config.json 版本号规范**：在配置文件顶级增加 `version` 字段（如 `"version": 2`），每次大版本升级时递增。

### 12.7 多用户与公共桌面权限

| 场景 | 处理策略 |
|------|----------|
| 标准用户运行 | 可正常读写用户桌面，公共桌面仅可读（可扫描但不可移动/隐藏其中的文件） |
| 管理员运行 | 完整权限，可读写用户桌面和公共桌面 |
| 提示提权 | 当操作涉及公共桌面写入时，弹窗说明需要管理员权限，提供"以管理员身份重启"选项 |
| UAC 拒绝 | 跳过公共桌面的写入操作，仅处理用户桌面的文件，Toast 提示"公共桌面操作已跳过" |

### 12.8 国际化预留

- 当前版本硬编码中文界面（简体）
- 架构预留：所有用户可见字符串集中在 `src/renderer/app.js` 和 `src/desktop-box/app.js` 的顶部常量区域，便于后续替换为 i18n 调用
- **HTML 模板中的硬编码文字也需纳入预留范围**：包括按钮 `title` 属性、`placeholder` 属性、Toast 消息模板中的动态部分（如"已归类 {N} 个项目到 {M} 个盒子"）、确认对话框文案等。这些文字分散在 `index.html` 和 `app.js` 的 DOM 操作中，国际化时容易遗漏
- 未来版本（v3.0+）可引入轻量 i18n 方案（如自研 JSON 语言包），支持中文/英文切换
- 时间格式化函数 `formatRelativeTime()` 已使用中文 locale，国际化时需同步调整
